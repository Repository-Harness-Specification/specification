# Governance

The project is currently in an early maintainer-led phase.

## Current model

- Maintainers merge changes after public review.
- Major design changes use the RFC process.
- Compatibility and benchmark evidence should guide decisions.
- Experimental features may be rejected when they expand the core without clear value.

## Future evolution

If the project gains regular contributors, governance may evolve to include:

- additional maintainers;
- documented voting rules;
- working groups for specification, tooling, benchmarks, and integrations;
- a formal release process.

Governance changes must be proposed publicly.
