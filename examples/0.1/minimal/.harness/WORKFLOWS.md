# Workflows

## Feature

1. Confirm the expected behavior and acceptance criteria.
2. Identify the capability or module that owns the change.
3. Make the smallest coherent implementation.
4. Add or update tests.
5. Run required validation.
6. Summarize changed files, decisions, and remaining manual checks.

## Bugfix

1. Reproduce or characterize the failure.
2. Add a regression test when practical.
3. Fix the narrowest responsible behavior.
4. Run related tests and full required validation.
5. Report the root cause and evidence.

## Refactoring

1. Preserve observable behavior.
2. Avoid unrelated feature changes.
3. Keep or improve test coverage.
4. Run full required validation.
