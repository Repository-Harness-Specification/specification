# Roadmap

## Phase 1: Problem definition

- Publish the initial problem statement.
- Define Repository Harness terminology.
- Create the draft specification repository.

## Phase 2: Manual implementation

- Migrate one real repository from a monolithic `AGENTS.md`.
- Record ambiguities and missing capabilities.
- Refine the minimal manifest.

## Phase 3: Benchmark

- Create reproducible implementation tasks.
- Compare monolithic and modular repository context.
- Measure task success, validation results, token usage when available, execution time, correction cycles, and out-of-scope changes.

## Phase 4: Tooling

- Build `harness-me init`.
- Build `harness-me validate`.
- Build `harness-me audit`.
- Generate entry-point adapters only after the source format stabilizes.

## Phase 5: Ecosystem

- Publish reusable Agent Skills.
- Add community-maintained examples.
- Accept third-party benchmark results.
- Evaluate a stable `1.0` specification.
